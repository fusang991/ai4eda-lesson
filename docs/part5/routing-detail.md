# 5.1 芯片绕线Routing过程

## 芯片绕线Routing过程——从Global Routing到Detailed Routing

### 基本概念

芯片布线是在布局（Placement）和时钟树综合（CTS）之后，将所有需要连接的引脚用金属线连接起来的过程。

### 布线资源

- **金属层（Metal Layer）**：现代工艺有10-15层金属
- **走线（Track）**：每层金属上的走线轨道
- **间距（Pitch）**：相邻走线之间的距离
- **通孔（Via）**：连接不同金属层的结构

```
Metal 3: ──────────────────────
Via 2-3: |
Metal 2: ──────────────
Via 1-2: |
Metal 1: ──────
         |
        标准单元引脚
```

### 芯片布线的优化目标

| 目标 | 说明 |
|------|------|
| 总线长 | 最小化总金属线长度 |
| DRC违规 | 零DRC违规（可制造性） |
| 信号完整性（SI） | 避免串扰和噪声 |
| 时序（Timing） | 满足时序约束 |
| PPA | 功耗、性能、面积的平衡 |

## Global Routing——全局布线

### G-cell网格

将芯片划分为粗粒度的网格单元（G-cell）：

```
+----+----+----+----+
| G1 | G2 | G3 | G4 |
+----+----+----+----+
| G5 | G6 | G7 | G8 |
+----+----+----+----+
| G9 |G10 |G11 |G12 |
+----+----+----+----+
```

### 全局布线任务

对于每个网络（Net），确定它经过哪些G-cell：

```
Net1: G1 -> G2 -> G6 -> G10
Net2: G3 -> G7 -> G11 -> G12
...
```

### 全局布线的优化

- 评估每个G-cell的布线拥塞度
- 避免过度拥塞的区域
- 最小化总线长

## Detailed Routing——详细布线

### 详细布线任务

在全局布线的指导下，在每个G-cell内部完成实际的金属连线：

```
全局布线: G1 -> G2 -> G6
详细布线: 具体的走线坐标和金属层分配
  Track 1: 从(100, 200)沿Metal2走到(300, 200)
  Via 2-1: 在(300, 200)换到Metal1
  Track 2: 从(300, 200)沿Metal1走到(300, 400)
```

### 设计规则检查（DRC）

详细布线必须满足所有DRC规则：

```
1. 最小宽度: 金属线宽度 >= Wmin
2. 最小间距: 相邻金属线间距 >= Smin
3. 最小面积: 金属线面积 >= Amin
4. Via规则: Via大小、Via间距等
5. 天线效应: 避免长金属线连接到栅极
```

### 信号完整性（Signal Integrity）

- **串扰（Crosstalk）**：相邻走线之间的电磁耦合
- **IR Drop**：电源网络上的电压降
- **EM（Electromigration）**：高电流密度导致的金属迁移

## 布线流程总结

```
输入: CTS后的网表（含时钟树）
    |
    v
1. Global Routing
   - 划分G-cell网格
   - 为每个网络规划G-cell路径
   - 优化拥塞度
    |
    v
2. Track Assignment
   - 将全局布线的路径分配到具体的Track
    |
    v
3. Detailed Routing
   - 在每个G-cell内完成详细走线
   - 修复DRC违规
    |
    v
4. Post-Route优化
   - 时序优化
   - SI优化
   - DRC修复
    |
    v
输出: 布线完成的版图
```
