# 4.1 时钟树基本概念

## 片上时钟和时钟树简介

### 时钟的作用

时钟是数字电路的"心跳"，同步所有寄存器的操作。一个SoC芯片可能有数百个时钟域，驱动数百万个寄存器。

### 时钟的产生

- **PLL（锁相环）**：从参考时钟生成高频时钟
- **时钟分频器**：降低时钟频率
- **时钟多路选择器**：在多个时钟源之间切换

### 时钟树的基本概念

时钟树（Clock Tree）是从时钟源到所有寄存器的分配网络。它的目标是让时钟信号同时到达所有寄存器。

```
时钟源 (Clock Source)
    |
    v
时钟树根 (Clock Root)
    |
    +-- Buffer --+-- Buffer --+-- FF1
    |            |            +-- FF2
    |            +-- FF3
    +-- Buffer --+-- FF4
                 +-- FF5
```

## 关键指标

### 时钟偏斜（Clock Skew）

同一时钟沿到达不同寄存器的时间差：

```
Skew = max(到达时间) - min(到达时间)

理想情况: Skew = 0（所有寄存器同时收到时钟）
```

Skew影响时序：
- **正偏斜**（发送端先到）：有利于Setup，不利于Hold
- **负偏斜**（接收端先到）：有利于Hold，不利于Setup

### 时钟延迟（Clock Latency）

从时钟源到寄存器的总延迟：

```
Latency = 插入延迟(Insertion Latency) + 传播延迟(Source Latency)
```

过大的Latency会：
- 增加时钟周期中可用于逻辑计算的时间比例下降
- 增加对OCV（片上变化）的敏感性

### 时钟树的目标

| 指标 | 目标 |
|------|------|
| Skew | 最小化（接近0） |
| Latency | 控制在合理范围 |
| DRC | 满足设计规则（最大转换时间、最大扇出） |
| 功耗 | 时钟网络功耗最小化 |

## 时钟树的结构

### H树（H-Tree）

对称的H形结构，天然保证等长：

```
        ┌──────────────────────┐
        │          H           │
        │    ┌─────┴─────┐     │
        │    │           │     │
        └────┤           ├─────┘
             │           │
         ┌───┤           ├───┐
         │   │           │   │
         └───┘           └───┘
```

- 优点：等长、Skew小
- 缺点：不灵活、面积大、只适合规则布局

### 树形结构（Tree）

最常用的实际结构，通过Buffer插入控制延迟和Skew。

### 网格结构（Mesh）

在高性能设计中使用，将时钟网络做成网格：

```
+---+---+---+---+
|   |   |   |   |
+---+---+---+---+
|   |   |   |   |
+---+---+---+---+
```

- 优点：极低Skew
- 缺点：功耗大、面积大
